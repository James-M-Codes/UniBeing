Some useful information from Week 5A:

  1. Elements that belong to a form are nested inside the form element.
  2. Action is a required attribute for forms. For example, I could add a required tag for a first and last name on a form.
  3. The method attribute specifies the HTTP protocol for submitting data to the server. This is either HTTP or HTTPS.
  4. Input elements have type attributes, which are used to specify the type of control to display, and the type of data that can be inputted. A button is a example of a input element.
  5. Input types are associated with specific controls.
  6. Text is used for the input of any text. These are used for text fields on form pages.
  7. Checkboxes are used to select zero or more options. For example, I could tick where I lived, and what my nationality was.
  8. Radio buttons are used to select only one option from a list. Picking I like dogs is a example of a radio button.
  9. Using the name attribute for radios allows users to have multiple radio buttons with the same name, and it allows for only one choice at a time.
  10. Submit uses a button control. The button has no default behaviour.
  11. Reset is used to reset a form to the default state.
  12. The most recent HTML version, HTML5, added 13 new form input types, such as date, color, number and range.
  13. The attribute required tells the user that the field needs to be filled in.
  14. To help users fill in forms, placeholders should be used.
  15. Fieldsets, legends and labels improves accessibility and usability.
  16. Different platforms style form elements differently.
  17. UX stands for user experience
  18. Labels are a inline element.

Some useful information from Week 5B:

  1. Most users have a version of Chrome, followed by Safari, Edge (previously Internet Explorer) and Firefox / Other browsers.
  2. HTML should be well-written, consistent, indented and commented.
  3. CSS that is assigned to a property in HTML will be inherited by that element's children.

Some useful information from Week 3A:

  1. Cascades are used to determine which rules apply to which element, and how to handle conflicts between any rules. This means that seperate stylesheets will always be overriden by newer versions.
  2. Descendant selectors are used to match navigation with different pages.
  3. The CSS box model controls how elements are displayed and positioned on a web page. An example could be displaying a image, with text around it, clearly seperated with boxes.
  4. Elements are usually block level, which usually takes up a whole line on a webpage.
  5. Other elements can usually be inline with the block level, which are nested within a block element. This means the the block level is the parent, while the inline level is the child.
  6. Boxes can be organised into sequences, or they can be nested inside other boxes.
  7. Boxes have four properties: margin, border, padding and content
  8. If the box-sizing of a box is the default value, the following CSS properties controls the content-box size: width, min-width, max-width, height, min-height and max-height.
  9. Padding is used to create space inside of elements. This sits between the content and the edge of the padding of the box model.
  10. Margins are used to create space outside of the element, while still residing inside of the box.
  11. Width and height refer to the content area of a box.
  12. Margin and padding increase the total size of a box. They also increase the distance between a text element, and a image element for example, allowing users to be able to read them more easily.
  13. Height properties are not often defined.
  14. Each side of a box can be defined by using: padding-top, padding-right, padding-bottom and padding-left. This is not necessary if the size does not change.
  15. Similarly, I can also use the margin-top, margin-right, margin-bottom and margin-left properties.
  16. The default value of the background-color or background image elements is transparent. This means that the background colour is white, unless it is changed.
  17. The background image element applies a specified image type. They can be any supported image type on computers and browsers, but the most common are: .JPG, .PNG, .SVG and rarely, .TIF
  18. Small images can use the background-repeat: repeat; element, but larger images should not use this property, as it could lead to problems.
  19. The following can be used to stylise the background properties: background, background-color, background-image, background-repeat and background-position.
  20. Absolute units are fixed, whereas relative units are not, meaning they are flexible. Relative units are relative to a value specified for another property type. Relative units should be used more, if the website is supposed to adapt to different screen sizes.
  21. Pixel, %, rem, em, vw and vh are the most commonly used relative units in web development.
  22. em is the most scaleable relative length unit. This can be used in media queries later on.
  23. The default font size for browsers is usually 16 pixels. Some browsers may use different font sizes.
  24. The root element of a web document is the html element at the top. This can be thought of as a container for the entire webpage.

Some useful information from Week 3B:

  1. Mobile devices have driven a change in web development, with more developers using and targeting mobile devices for their websites. Different screen sizes allows developers to see what the website will look on a phone, a tablet, or a laptop / desktop.
  2. Responsive web design consists of three core responsive web design techniques: Fluid, CSS media queries, and responsive media.
  3. The viewport is equal to the width of the browser window. This is used to scale webpages to fit on smaller, or larger screens than the device it was originally written on.
  4. width=device-width sets the pixel width of the viewport to be equivalent to the screen width of a device.
  5. initial-scale tells the browser to render a web page at full size, as opposed to scaling the web page down.
  6. CSS media queries are the core mechanic on what makes web pages responsive. They inspect the capability of the device, by checking for: The width and height of the viewport, the target device screen width, the device height, the device's orientation of the screen, screen resolution and aspect ratio.
  7. Media queries usually consists of media types, and one or more expressions that checks for media features such as width, height and screen resolution.
  8. Media queries are booleans that are either true or false.
  9. Media queries are usaully true if the media type matches the device of the user, and if all of the expressions in the media query are true.
  10. The media screen CSS element checks whether the media type of the device accessing the screen meets the right pixel requirement.
  11. Expressions are written in brackets.
  12. The min-width CSS element tests the width of the viewport, to find out if the width is equal or greater than the specified pixels.
  13. CSS style rules are nested inside of the query in curly brackets. Custom rules can be set for different elements, by placing a dot at the beginning of the query. For example, writing .buttons, would target the HTML element class of "buttons", and it would then apply the CSS rule to that class.
  14. @Media targets different media types. Examples include: tablets, Personal computers, consoles, and phones.
  15. The term min means something is equal or greater than, whereas the term max means something is equal or smaller than
  16. Standard widths are used as breakpoints. Breakpoints are used to identify and fix bugs in websites that would otherwise slip under the net.

Some useful information:

  1. The World Wide Web was invented in 1989 by Sir Tim Berners-Lee, who is a British scientist
  2. In 1993, the early internet had different competing standards. Some browsers used different standards over others, which led to users being forced to download new browsers just to navigate different websites.
  3. In 1994, a solution was found to this problem: The World Wide Web Consortium was developed, which mandated that developers needed to have a unified system of accessing websites, thereby ending the need for multiple browsers.
  4. HTML is a programming language that is written for both users and browsers
  5. HTML as a language contains syntax and semantics
  6. HTML syntax consists of tags that are usually lowercase
  7. HTML semantics consists of tags that are marked up in a meaningful way. For example, the p element is a semantic.
  8. HTML markup specifies the structure of a web content in a meaningful way
  9. Cascading Style Sheets (or CSS) is how web browsers displays content, on a website. They are also used for accessibility purposes. One example of this is making font larger, changing font colour or changing the font size.
